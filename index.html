Modal vs Popup

Dos componentes fundamentales del diseño de interfaces. Parecidos visualmente, pero con propósitos, comportamientos y casos de uso completamente distintos.

▶ Ver ejemplo Comparar diferencias
¿Qué es cada uno?
⬜

Modal

Un modal (o diálogo modal) es una ventana que aparece sobre el contenido principal de la página, bloqueando toda interacción con el fondo hasta que el usuario realice una acción explícita.

  • Crea un overlay oscuro que cubre el fondo
  • El foco del teclado queda atrapado dentro
  • Se centra en la pantalla, sin anclarse a ningún elemento
  • Requiere acción del usuario para cerrarse
  • Interrumpe intencionalmente el flujo de trabajo
💬

Popup

Un popup es un elemento flotante que aparece anclado a un elemento específico de la página. No bloquea la interfaz y permite al usuario seguir interactuando con el resto del contenido.

  • No genera overlay ni oscurece el fondo
  • El foco puede salir libremente del componente
  • Aparece posicionado relativo a su elemento origen
  • Se cierra al hacer clic afuera o perder el hover
  • Es complementario al flujo, no lo interrumpe
Diferencias técnicas
Característica 🟣 Modal 🟢 Popup
Overlay de fondo Sí — oscurece toda la página No — el fondo permanece activo
Bloqueo de UI Total (foco atrapado) Ninguno — libre navegación
Posición Centrado en la pantalla Relativo al elemento disparador
Cómo se cierra Botón, Esc, clic en overlay Clic afuera, hover-out, Esc
Peso visual Alto — protagonista total Bajo — soporte al contenido
Interrupción Alta — intencional Muy baja — no invasivo
Tamaño típico Mediano a grande (300–600px) Pequeño a mediano (150–300px)
Nivel de urgencia Decisiones críticas Información contextual

¿Cuándo usar cada uno?
⬜

Usa un Modal cuando…

  • Necesitas confirmar una acción destructiva (eliminar, desactivar)
  • El usuario debe completar un formulario antes de continuar
  • Inicio de sesión / registro de usuario
  • Términos y condiciones que requieren aceptación explícita
  • Mostrar una alerta de error crítico o advertencia de seguridad
  • Previsualización ampliada de imágenes o media
  • Subir o seleccionar archivos importantes
💬

Usa un Popup cuando…

  • Quieres mostrar un tooltip con información de ayuda contextual
  • Menú desplegable de navegación o menú contextual
  • Selector de fecha (datepicker) o selector de color
  • Notificación breve no bloqueante (toast)
  • Vista previa de perfil de usuario al hacer hover
  • Mostrar opciones adicionales sin abandonar la vista actual
  • Formulario inline pequeño (buscar, filtrar, etiquetar)
Código básico
HTML · estructura del modal 
<!-- Botón que abre el modal -->
<button onclick="abrirModal()">Abrir Modal</button>

<!-- Overlay + Modal -->
<div id="overlay" class="overlay">
  <div
    class="modal"
    role="dialog"
    aria-modal="true"
    aria-labelledby="modal-titulo"
  >
    <h2 id="modal-titulo">Título del Modal</h2>
    <p>Contenido importante aquí.</p>

    <button onclick="cerrarModal()">Aceptar</button>
    <button onclick="cerrarModal()">Cancelar</button>
  </div>
</div>
HTML · estructura del popup 
<!-- Contenedor relativo al botón -->
<div class="popup-wrapper">

  <!-- Botón disparador -->
  <button onclick="togglePopup()">
    Más info
  </button>

  <!-- Popup anclado al botón -->
  <div
    id="popup"
    class="popup hidden"
    role="tooltip"
  >
    <h4>Información</h4>
    <p>Aquí va la info contextual.</p>
  </div>

</div>
CSS · estilos del modal y overlay 
/* Overlay que oscurece el fondo */
.overlay {
  display: none;
  position: fixed;  /* cubre toda la pantalla */
  inset: 0;
  background: rgba(0, 0, 0, 0.75);
  z-index: 9999;
  align-items: center;
  justify-content: center;
}

.overlay.activo {
  display: flex;  /* se muestra */
}

/* Caja del modal centrada */
.modal {
  background: white;
  border-radius: 12px;
  padding: 2rem;
  max-width: 480px;
  width: 90%;
}

/* Popup — posición relativa al padre */
.popup {
  position: absolute;
  bottom: calc(100% + 10px);
  left: 50%;
  transform: translateX(-50%);
}

.popup.hidden { display: none; }
JavaScript · comportamiento 
// ── MODAL ──────────────────────────
const overlay = document.getElementById('overlay');

function abrirModal() {
  overlay.classList.add('activo');
  // Trampa de foco para accesibilidad
  document.body.style.overflow = 'hidden';
}

function cerrarModal() {
  overlay.classList.remove('activo');
  document.body.style.overflow = '';
}

// Cerrar con la tecla Escape
document.addEventListener('keydown', (e) => {
  if (e.key === 'Escape') cerrarModal();
});

// ── POPUP ───────────────────────────
const popup = document.getElementById('popup');

function togglePopup(event) {
  event.stopPropagation();  // evita cierre inmediato
  popup.classList.toggle('hidden');
}

// Cerrar al hacer clic fuera
document.addEventListener('click', () => {
  popup.classList.add('hidden');
});
manera de usar

Si el usuario DEBE actuar antes de continuar → usa un Modal.
Si solo quieres mostrar info extra sin interrumpir → usa un Popup.
